## Explanation

The problem asks us to minimize the sum of $|a_i - x|$ for $i\in[0\ldots n)$ and
an $x$ of our choice under modulus $M$. Thus, let us first consider this problem
without the modulus.

Letting $x$ be the _median_ of those numbers minimizes the total distance. In
the case that $N$ is even, any number between the two numbers in the middle
(inclusive) would work.

<Spoiler title="Intuition on this property">

Let $m_1=m_2=a_{\lfloor N/2\rfloor +1}$ if $N$ is odd; otherwise, let
$m_1=a_{N/2}$ and $m_2=a_{N/2+1}$. There are an equal number of indices to the
left and right of $[m_1, m_2]$, so anything in $[m_1, m_2]$ should be optimal.

For the sake of contradiction, suppose $x$ is outside the range.

If we increment $x$ by $1$, its distance from all $a_i > x$ will decrease by
$1$, and distances from all $a_i < x$ will increase by $1$. Since $x$ is to the
left of $[m_1, m_2]$, there are more $a_i > x$ than $a_i < x$. Thus, moving $x$
closer to $m_1$ decreases the total difference.

Using similar logic for $x>m_2$ gives that the range $[m_1, m_2]$ is optimal.

</Spoiler>

Now, we consider the problem when there is a modulus, starting by modding each
number by $M$. Under a modulus, _any_ number can be made a median by subtracting
$M$ from a suitable amount of the largest elements or, conversely, by adding $M$
to a suitable amount of the smallest elements. For example, given
$[1,3,7,9,13,17,19]$ and $M = 24$, we can make $17$ the median by adding $M$ to
$1$ and $3$, resulting in $[7,9,13,17,19,25,27]$.

We prepend to $a$ a copy of $a$ with $M$ subtracted from every element, and
append a copy of $a$ with $M$ added to every element. The range $[n, 2n)$ in $a$
will be the original elements. For each of these indices, we compute the amount
of elements before and after it we need to select to make it a median, and then
compute the total absolute difference using Prefix Sums.

The final answer will be the smallest result over all such indices.

## Implementation

**Time Complexity:** $\mathcal{O}(N \log N)$ per test case

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

using namespace std;

void solve() {
	int n, m;
	cin >> n >> m;
	vector<long long> a;
	for (int i = 0; i < n; i++) {
		long long value;
		cin >> value;
		value %= m;
		// The first n elements are - m.
		value -= m;
		a.push_back(value);
	}
	sort(a.begin(), a.end());
	// The middle elements are the original modded elements, so add back m.
	for (int i = 0; i < n; i++) a.push_back(a[i] + m);
	// Last elements are +m
	for (int i = 0; i < n; i++) a.push_back(a[i] + 2 * m);

	vector<long long> prefix_sum;
	prefix_sum.push_back(a[0]);
	for (int i = 1; i < 3 * n; i++) prefix_sum.push_back(prefix_sum[i - 1] + a[i]);

	long long ans = 9e18;
	for (int i = n; i < 2 * n; i++) {
		/*
		 * Compute minimum total difference from each a[i] to the x.
		 * We treat "median" as the left median if n is even, and decompose
		 * into left and right calculations (since absolute value argument
		 * sign changes)
		 */
		long long cost = 0;

		// Compute cost for difference between all elements on
		// the left (smaller) to the current value
		if (n % 2 == 0) {
			/*
			 * If n is even, then sum for the next n/2 - 1 elements.
			 * This is sum of (a[j] - a[i]) from j = i-(n/2-1) to j=i-1.
			 * The a[i] gets factored out of the sum, and results in the
			 * below:
			 */
			cost += a[i] * (n / 2 - 1) - (prefix_sum[i - 1] - prefix_sum[i - n / 2]);
		} else {
			// For n/2 elements
			// This is sum of (a[j] - a[i]) from j = i-(n/2) to j=i-1.
			cost += a[i] * (n / 2) - (prefix_sum[i - 1] - prefix_sum[i - n / 2 - 1]);
		}

		// Right side of the current value (larger).  Always n/2 elements
		// to the right. Sum of a[j] - a[i] from j=i+1 to j=i+n/2.
		cost += (prefix_sum[i + n / 2] - prefix_sum[i]) - a[i] * (n / 2);
		ans = min(ans, cost);
	}
	cout << ans << endl;
}

int main() {
	int test_num;
	cin >> test_num;
	for (int t = 0; t < test_num; t++) solve();
	return 0;
}
```

```py
T = int(input())
for _ in range(T):
	N, M = map(int, input().split())
	nums = list(map(int, input().split()))
	# Mod, sort, and construct array
	rems = sorted([x % M for x in nums])
	# S is prepended/appended array
	S = [x - M for x in rems] + rems + [x + M for x in rems]

	# prefix sums for fast lookup of operations needed
	prefix = [S[0]]
	for i in range(1, len(S)):
		prefix.append(prefix[i - 1] + S[i])

	min_total = float("inf")

	for i in range(N, 2 * N):

		# Compute minimum total difference from each a[i] to the x.
		# We treat "median" as the left median if N is even, and decompose
		# into left and right calculations (since absolute value argument
		# sign changes)

		cost = 0

		# Compute cost for difference between all elements on
        # the left (smaller) to the current value
		if N % 2 == 0:

			# If N is even, then sum for the next N//2 - 1 elements.
			# This is sum of (S[j] - S[i]) from j = i-(N//2-1) to j=i-1.
			# The S[i] gets factored out of the sum, and results in the
			# below:
        
			cost += S[i] * (N // 2 - 1) - (prefix[i - 1] - prefix[i - N // 2])
		else:
			# If N is odd, then do for N//2 elements:
			# This is sum of (S[j] - S[i]) from j = i-(N//2) to j=i-1.
			cost += S[i] * (N // 2) - (prefix[i - 1] - prefix[i - N // 2 - 1])

		# Right side of the current value (larger).  Always N//2 elements
		# to the right. Sum of a[j] - a[i] from j=i+1 to j=i+N//2.
		cost += (prefix[i + N // 2] - prefix[i]) - S[i] * (N // 2)

		# Update min_total
		min_total = min(min_total, cost)

	print(min_total)
```
