## Explanation

The problem asks us to minimize the sum of $|a_i - x|$ for all $a_i$, using an $x$ of our choice, with each difference under modulo $M$
(strictly, optimizing the sum of $\min(a_i - x \bmod M, x - a_i \bmod M)$).
Thus, let us first consider this problem without the modulus.

Letting $x$ be the _median_ of those numbers [minimizes the total distance](https://cses.fi/book/book.pdf#page=71). In
the case that $N$ is even, any number between the two numbers in the middle
(inclusive) would work.

We now consider the modulus. First, take each number modulo $M$. Observe that any element can be the median if we add or subtract $M$ from some elements, shifting some values to the left and right of the element. For example, given $[1,3,7,9,13,17,19]$ and $M = 24$, we can make $17$ the median by adding $M$ to elements $1$ and $3$, resulting in the array $[7,9,13,17,19,25,27]$.

Thus, every element is a candidate for $x$.
We compute the cost for each $x$ after shifting, and take the best candidate, which is the $x$ that results in the smallest sum of absolute differences.

To implement this, we first sort $a$.
Then, we consider every element in $a$ for a potential $x$, and compute the amount of elements before and after it that needs to be present to make it a median.
After that, we use prefix sums to compute the total absolute difference.
However, this requires us to access indices of $a$ that are not in the $[0, n)$ range;
to mitigate this, we prepend to $a$ a copy of $a$ with $M$ subtracted from every element, and
append a copy of $a$ with $M$ added to every element.
The range $[n, 2n)$ in $a$ will then be the original elements we enumerate over, and we will be able to access the elements that come before or after the original array.

The final answer will be the smallest sum of absolute differences over all such indices.

## Implementation

**Time Complexity:** $\mathcal{O}(N \log N)$ per test case

```cpp
#include <algorithm>
#include <climits>
#include <iostream>
#include <vector>

using namespace std;

void solve() {
	int n, m;
	cin >> n >> m;
	vector<long long> a;
	for (int i = 0; i < n; i++) {
		long long value;
		cin >> value;
		value %= m;
		// The first n elements are - m.
		value -= m;
		a.push_back(value);
	}
	sort(a.begin(), a.end());
	// The middle elements are the original modded elements, so add back m.
	for (int i = 0; i < n; i++) { a.push_back(a[i] + m); }
	// Last elements are +m
	for (int i = 0; i < n; i++) { a.push_back(a[i] + 2 * m); }

	vector<long long> prefix_sum;
	prefix_sum.push_back(a[0]);
	for (int i = 1; i < 3 * n; i++) { prefix_sum.push_back(prefix_sum[i - 1] + a[i]); }

	long long ans = LLONG_MAX;
	for (int i = n; i < 2 * n; i++) {
		/*
		 * Compute minimum total difference from each a[i] to the x.
		 * We treat "median" as the left median if n is even, and decompose
		 * into left and right calculations (since absolute value argument
		 * sign changes)
		 */
		long long cost = 0;

		// Compute cost for difference between all elements on
		// the left (smaller) to the current value
		if (n % 2 == 0) {
			/*
			 * If n is even, then sum for the next n/2 - 1 elements.
			 * This is sum of (a[j] - a[i]) from j = i-(n/2-1) to j=i-1.
			 * The a[i] gets factored out of the sum, and results in the
			 * below:
			 */
			cost += a[i] * (n / 2 - 1) - (prefix_sum[i - 1] - prefix_sum[i - n / 2]);
		} else {
			// For n/2 elements
			// This is sum of (a[j] - a[i]) from j = i-(n/2) to j=i-1.
			cost += a[i] * (n / 2) - (prefix_sum[i - 1] - prefix_sum[i - n / 2 - 1]);
		}

		// Right side of the current value (larger).  Always n/2 elements
		// to the right. Sum of a[j] - a[i] from j=i+1 to j=i+n/2.
		cost += (prefix_sum[i + n / 2] - prefix_sum[i]) - a[i] * (n / 2);
		ans = min(ans, cost);
	}
	cout << ans << endl;
}

int main() {
	int test_num;
	cin >> test_num;
	for (int t = 0; t < test_num; t++) { solve(); }
	return 0;
}
```

```py
t = int(input())
for _ in range(t):
	n, m = map(int, input().split())
	nums = list(map(int, input().split()))

	# Mod, sort, and construct array
	rems = sorted([x % m for x in nums])

	# s is prepended/appended array
	s = [x - m for x in rems] + rems + [x + m for x in rems]

	# prefix sums for fast lookup of operations needed
	prefix = [s[0]]
	for i in range(1, len(s)):
		prefix.append(prefix[i - 1] + s[i])

	min_total = float("inf")

	for i in range(n, 2 * n):
		# Compute minimum total difference from each s[i] to the x.
		# We treat "median" as the left median if n is even, and decompose
		# into left and right calculations (since absolute value argument
		# sign changes)
		cost = 0

		# Compute cost for difference between all elements on
		# the left (smaller) to the current value
		if n % 2 == 0:
			# If n is even, then sum for the next n//2 - 1 elements.
			# This is sum of (s[j] - s[i]) from j = i-(n//2-1) to j=i-1.
			# The s[i] gets factored out of the sum, and results in the
			# below:
			cost += s[i] * (n // 2 - 1) - (prefix[i - 1] - prefix[i - n // 2])
		else:
			# If n is odd, then do for n//2 elements:
			# This is sum of (s[j] - s[i]) from j = i-(n//2) to j=i-1.
			cost += s[i] * (n // 2) - (prefix[i - 1] - prefix[i - n // 2 - 1])

		# Right side of the current value (larger). Always n//2 elements
		# to the right. Sum of s[j] - s[i] from j=i+1 to j=i+n//2.
		cost += (prefix[i + n // 2] - prefix[i]) - s[i] * (n // 2)

		# Update min_total
		min_total = min(min_total, cost)

	print(min_total)
```
