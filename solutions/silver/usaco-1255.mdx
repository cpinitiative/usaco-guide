## Explanation

Below, $a_i$ broadly refers to the current number of cows in the $i$-th room, rather than simply the initial number;
$\mathbb P$ is used to refer to the set of primes; for this problem, it includes $1$.

First, let's approach this problem when $N=1$, which is when when both farmers remove cows from one room until it is empty.

Notice that it is possible to remove $1$, $2$, or $3$ but not any multipes of $4$ cows.
This implies that we should try to center our strategy around manipulating $a_1\bmod 4$.
First, if a player arrives in a state where $a_1\bmod 4=0$, then they cannot get out of it, as there are no multiples of $4$.
On the other hand, if a player arrives in a state where $a_1\bmod 4\neq 0$, they could always take away $a_1\bmod 4\in\{1,2,3\}$ to force the other player into a state where $a_1\bmod 4=0$, which they cannot get out of.

Our strategy for $N=1$ thus falls out immediately:
A player with a state of $a_1\bmod 4\neq 0$ should always take away $a_1\bmod 4$ cows to force the other player into a state where $a_1\bmod 4=0$;
that player will always return with a state of $a_1\bmod 4\neq 0$, and you can repeat this and give them a smaller $a_1\bmod 4=0$, which will eventually choke them down into $0$, which is when you win.
Therefore, when the initial state is $a_1\bmod 4\neq 0$, Farmer John will win;
but if $a_1\bmod 4=0$ initially, then Farmer John must produce a state of $a_1\bmod 4\neq 0$ for Farmer Nhoj, forcing Farmer Nhoj to win instead.

Now, we consider the case when $N>1$.  The farmers will go in circles, each playing each room only once; however, the operations on each room are all independent.
This means that we can consider the result of playing each room as if $N=1$, and term guaranteed loser of that room the "defender" and the guaranteed winner of that room the "attacker".

The farmers will play in a circle, so the next round of operations on a room will only be done when all the other rooms are visited.
Therefore, the attacker in a room would like to get to 0 as fast as possible in that grid, and the defender would like the room to be played slower instead.
This is because the attacker wants to minimize the chances that he gets beaten at another room, and therefore needs to minimize the number of times he goes around the circle before coming back, so he can hopefully win right here early;
the defender would want to sabotage the attacker from doing so, and therefore needs to maximize the potential duration of playing at this room.
When two players play with the above strategies, let the number of total operations on room $i$ if it is played to the end be $T_i$;
the game's winner will be the winner of the room with the minimum $\left\lceil\frac{T_i}2\right\rceil$ (if tied, the minimum room number), as it's the room that will be reached after the least amount of times went around the circle.

To minimize the number of rounds, the attacker can use a slightly modified version of his strategy when $N=1$: 
Instead of giving $a_i\bmod4=0$'s to the other victim by taking away $a_i\bmod 4$ cows, he can take away any $p\equiv a_i\mod 4$ cows, where $p \in \mathbb P$;
the larger the $p$ for the operation, the more speedup it offers to ending the game.
Given an $x$, to compute the largest $p$ for which $p\equiv x\mod 4$, one preprocesses;
first, use a sieve to find all primes, and in the process, keep an array $t_i$ with $0\leq i<4$ for the most recent prime $t_i=p$ we found where where $p\bmod 4=i$;
then, we assign $m_x=t_{x\bmod 4}$ to keep track of the largest prime such that $m_x\equiv x\mod 4$.

On the other hand, the defender can block this attack as soon as they receive an $a_i\bmod 4=0$.
This relies on the fact that there are no primes $p>4$ that satisfies $p\bmod 4=2$, and there are no speedups when $a_i\bmod 4=2$;
therefore, if the defender takes $2$ cows, and hand back $a_i\bmod 4=2$, and do this consistently, there will be no way for the attacker to speed up the game any more.
This means that the only speedup the attacker can apply is when $a_i\bmod4\in\{1,3\}$ (which also implies he goes first), removing $m_{a_i}$ initially;
After that, or in other cases, either there are no speedups available, or the defender will block all of them.
When there are no speedups, there are obviously $T_i=\frac{a_i}2$ operations only (in which $T_i$ represents the number of turns at this room);
when there is a speedup, there are $T_i=1+\frac{a_i-m_{a_i}}2$, as one operations is used to remove $m_{a_i}$.

We first compute the amount of operations ($T_i$) for each room and determine the number of cows in it $\bmod 4$ to determine the winner for that room;
we then find the argmin of $\left\lceil\frac{T_i}2\right\rceil$ (if tied, with smallest $i$), and output the winner of that room.
In practice, however, it becomes unneccessary to track the room, as the parity of the number of operations already encodes the winner.


## Implementation

**Time Complexity:** $\mathcal{O}(N)$

```cpp
#include <iostream>
#include <cstring>
using namespace std;
constexpr int MAX_NUMBER = 5000000;
int isprime[MAX_NUMBER+1];
int m[MAX_NUMBER+1];

int main()
{
    int T;
    cin >> T;
    memset(isprime, true, sizeof isprime);
    isprime[1] = false;
	// 1 is considered a prime under this problem, even though
	// we don't sieve it; so assign 1 initially.
    int t[] = {-1, 1, -1, -1};
	m[1] = 1;
	// Sieve algorithm
    for (int i = 2; i <= MAX_NUMBER; i++) {
        if (isprime[i]) {
            t[i%4] = i;
            for (int j = i; j < MAX_NUMBER; j += i)
                isprime[j] = false;
        }
		m[i] = t[i % 4];
    }
    while (T--) {
        int n;
        cin >> n;
        int minn = 2e9;
        for (int i = 1; i <= n; i++) {
            int a;
            cin >> a;
			int min_turns;
			if (a % 2 == 0) {
				min_turns = a / 2;
			} else {
				min_turns = 1 + (a - m[a]) / 2;
			}
            if (min_turns / 2 < minn / 2) {
                minn = min_turns;
            }
        }
        cout << ((minn & 1) ? "Farmer John" : "Farmer Nhoj") << endl;
    }
    return 0;
}
```
