## Explanation

Firstly, let's consider solving this problem in 1D, where we pick intervals from the list $g(j)$, the values for an $1\times N$ pasture.

Let $g'(j)$ be $-1$ if $g(j)<100$, $0$ if $g(j)=100$, and $1$ if $g(j)>100$.
We must find a range of $j$'s in which there are no $-1$'s, and at least one $0$.
We precompute $g'(j)$.
The objective is now counting the number of intervals with no $-1$'s and at least one $0$.

Next, we sweep $j$ from $1$ to $N$, and keep track of the last index at which we encounter $0$ and $-1$'s as $j_0$ and $j_{-1}$, respectively.
For an interval to be picked, it must include a $0$, so the starting index must be no later than the last $0$ we encounter;
the interval cannot include a $-1$, so the starting index must be later than the last $-1$ we encounter.
Thus, any starting index in the range $(j_{-1}, j_0]$ works (provided $j_{-1}<j_0$), so we simply accumulate our current value of $\max(0, j_0-j_{-1})$ onto the answer if $g'(j)\in{0,1}$ (as we cannot end with $g'(j)=-1$) while we sweep.
We initialize both $j_0$ and $j_{-1}$ to $0$, so that they produce no accumulation initially, and that all possibilities of $(0, j_0]$ will be counted if a $-1$ has not occurred yet.

Now, we generalize this onto two dimensions, with an $N\times N$ pasture of $G(i, j)$.
First, we consider the cases where the rectangle starts at $i=1$ and ends at $i=N$ vertically.
Once the top and bottom rows are fixed, the only remaining choice is the left and right columns, so the problem reduces to counting valid intervals horizontally.
For $g'$, however, $g'(j)$ should be marked as $-1$ if any $i$ has $G(i, j)<100$, as any $-1$ blocks selection.
$g'(j)$ can be marked as $0$ if any $i$ has $G(i, j)=100$, as only one $100$ is needed to select a range.
This leaves $g'(j)$ as $1$ only if all $i$'s have $G(i, j)>100$.

We now generalize to the case where the rectangle can start at any $i=i_1$ and end at any $i=i_2\geq i_1$.
At each possible $i_1$ we enumerate, we reset $g'(j)$ to $1$ (as having no possible $i$'s satisfies "all $i$'s have $G(i, j)>100$"), before we loop over $i_2$ and $j$.
Since $g'(j)$ already accounts for $G(i_1\ldots i_2-1,j)$, we update it to explicitly to account for $G(i_2,j)$, the newly added cell, at each $j$ in the inner loop.
After the $g'(j)$ update, we proceed as we would in the $1$-dimensional case.
Tracking the array $g'$ cumulatively gives us $\mathcal O(N)$ time complexity for each choice of top and bottom rows, as previous $g'$s are reused to avoid recomputing each from scratch, which takes $\mathcal O(N^2)$ time.

## Implementation

**Time Complexity:** $\mathcal O(N^3)$

```cpp
#include <iostream>

using namespace std;

constexpr int MAX_N = 500;

// G in problem
int a[MAX_N + 1][MAX_N + 1];

// g' in explanation
int g_prime[MAX_N + 1];

int main() {
	int n;
	cin >> n;

	for (int i = 1; i <= n; i++)
		for (int j = 1; j <= n; j++) cin >> a[i][j];

	long long ans = 0;
	for (int i1 = 1; i1 <= n; i1++) {
		// Reset g'
		for (int j = 1; j <= n; j++) g_prime[j] = 1;

		for (int i2 = i1; i2 <= n; i2++) {
			// i_0 and i_{-1} in explanation, respectively
			int last_zero = 0, last_neg = 0;
			for (int j = 1; j <= n; j++) {
				// Update g'
				if (a[i2][j] < 100) g_prime[j] = -1;
				if (a[i2][j] == 100 && g_prime[j] != -1) g_prime[j] = 0;

				// Update last indices
				if (g_prime[j] == 0) last_zero = j;
				if (g_prime[j] == -1) last_neg = j;

				// Accumulate
				if (g_prime[j] == 0 || g_prime[j] == 1) {
					ans += max(0, last_zero - last_neg);
				}
			}
		}
	}

	cout << ans << endl;
	return 0;
}
```
