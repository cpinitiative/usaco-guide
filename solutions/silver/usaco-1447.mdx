## Explanation

We can approach this question as cutting down the maximum number of trees while satisfying all the restrictions.

First, maintain a list containing both the trees and the intervals, sorted by the intervals' left endpoint and the trees' positions. Then, we can traverse over this list from left to right.

Consider the case where we encounter an interval while procesing. Let $\text{cur}$ be the current number of trees cut down, which represents the current answer. When we add a new interval, it imposes a restriction on the amount of trees that can be cut down in that interval's range. Formally, if $\text{present}$ is the current amount of trees present in that interval, then at most $\text{cut} = \text{present} - t_i$ trees can be cut down in that interval's range. The value of the answer at the end of this interval is therefore at max $\text{cur} + \text{cut}$.

To maintain these conditions, we can use a priority queue that keeps track of the smallest amount of trees that can be cut at the given point in time. Whilst processing a tree, as long as the maximum amount of trees of the current intervals has not already been cut, we can greedily cut down the current tree.

It can be shown that this strategy is optimal. If we instead chose to cut down a future tree, it could affect both future and current intervals, potentially reducing the total number of trees we can cut. In contrast, cutting down the current tree only affects the current intervals, making it always the optimal choice.

## Implementation

**Time Complexity:** $\mathcal{O}((N + K)\log(N + K))$

#include <bits/stdc++.h>
using namespace std;

void solve() {
	int n, k;
	cin >> n >> k;
	vector<int> x(n);
	for (auto &d : x) cin >> d;

	sort(x.begin(), x.end());

	vector<array<int, 4>> critical_points;
	// Process intervals and insert them into the critical points list, using their
	// right and left bounds, and the amount of trees in the interval minus the
	// restriction
	for (int i = 0; i < k; i++) {
		int left, right, t;
		cin >> left >> right >> t;
		auto right_amt = upper_bound(x.begin(), x.end(), right);
		auto left_amt = lower_bound(x.begin(), x.end(), left);
		critical_points.push_back({l, 0, r, (right_amt - left_amt) - t});
	}

	// Add trees to the critical points list using their position
	for (int i = 0; i < n; i++) { critical_points.push_back({x[i], 1, 0, 0}); }

	// Sort the list containing both the trees and intervals
	sort(critical_points.begin(), critical_points.end());

	priority_queue<array<int, 2>, vector<array<int, 2>>, greater<array<int, 2>>> pq;

	int ans = 0;

	// Process all the critical points
	for (auto point : critical_points) {
		int type = point[1];

		if (type == 0) {
			// Interval Processing
			int right = point[2];
			int amt_trees = point[3];

			pq.push({ans + amt_trees, right});
		} else {
			// Tree Processing
			int tree_position = point[0];
			while (!pq.empty() and (pq.top())[1] < tree_position) pq.pop();

			if (pq.empty() or (pq.top())[0] != ans) ans++;
		}
	}

	cout << ans << '\n';
}

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(nullptr);

	int t;
	cin >> t;
	while (t--) { solve(); }
}

import sys

input = sys.stdin.readline
import heapq
import bisect


def solve():
	n, k = map(int, input().split())
	x = list(map(int, input().split()))
	x.sort()

	critical_points = []

	# Process intervals and insert them into the critical points list, using their right and left bounds, and the amount of trees in the interval minus the restriction
	for _ in range(k):
		left, right, t = map(int, input().split())
		right_amt = bisect.bisect_right(x, right)
		left_amt = bisect.bisect_left(x, left)
		critical_points.append([left, 0, right, (right_amt - left_amt) - t])

	# Add trees to the critical points list using their position
	for i in range(n):
		critical_points.append([x[i], 1, 0, 0])

	# Sort all critical points
	critical_points.sort()

	pq = []  # min-heap
	ans = 0

	# Process all critical points
	for point in critical_points:
		type_ = point[1]

		if type_ == 0:
			# Process Interval
			right = point[2]
			amt_trees = point[3]
			heapq.heappush(pq, (ans + amt_trees, right))
		else:
			# Process a Tree
			tree_position = point[0]

			while pq and pq[0][1] < tree_position:
				heapq.heappop(pq)

			if not pq or pq[0][0] != ans:
				ans += 1

	print(ans)


def main():
	t = int(input())
	for _ in range(t):
		solve()


if __name__ == "__main__":
	main()
