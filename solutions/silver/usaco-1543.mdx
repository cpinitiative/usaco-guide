# Solution

## Explanation

First, notice that the constraints $(x, y, z)$ form an undirected weighted graph, where the constraints
are the edges and the generators are the vertices.  Each connected component can be optimized independently,
and below we shall consider one connected component.

Let us first consider the cases when the component has no cycles. In this case, we choose an arbitary vertex,
$v$, and let the variable $x$ represent $a_v$.

Since there are no cycles, we may start from vertex $v$ and traverse all vertices in the connected component
without visiting any already visited vertices. In the traversal, since each edge is a constraint,
we can write the value for the node we go to in terms of the node we came from. For example, if the
traversal leads us from node $1$ to node $2$ with the constraint $a_1+a_2=3$, then we may write $a_2=3-a_1$.

We can build a representation of all other nodes in the current connected component in terms of $x$ representing
each node in terms of the value it was traversed from using the method described in the previvous paragraph.
We can then set this expression equal to
the minimum and maximum values for the generator to function, and solve for $x$ to find out what ranges of
$x$ is required for this specific generator to function. After this, we have multiple ranges of $x$, each
representing the range needed for each generator to function. From there, we must find a value of $x$ that
is in the maximum amount of ranges, which means that the maximal amount of generators can function.
We can use a difference array (using a map) to add 1 to the number of ranges there in each range, and then
loop over the map, keep track of the prefix sums of that array, and then max them out to obtain the
largest amount of generators that can function in this component.

Specifically, for implementation, we choose a node $v$ to start from in each connected comopnent (and name
this connected component Component $v$), and
represent $v$'s power as $x$, so the power will be $1x+0$, which is $m[v]=1$
and $b[v]=0$. Then, for each node we traverse to, if we're going from node $u$ to node $v$ through
constraint $a_u + a_v = c$, then we can represent $a_v=c-a_u=c-(m[u]x+b[u])=-m[u]x+(c-b[u])$, aka.
$m[v]=-m[u]$ and $b[v]=(c-b[u])$. We do this until we represent every node; we also set the
expression equal to the minimum and maximum values of each node, and solve; the lower value we get
is the minimum $x$, and the higher the maximum $x$ for the generator to function.

Now, what happens if we have cycles? In our traversal for each component, cycles break the assumption that
we will not revisit nodes. When we do revisit one, we will get *two* different representations of power
mandated for a generator. We may equate the expressions, and solve for $x$; if unsolvable, this problem
is unsolvable; if infinitely many solutions, then we don't need to do anything, same as if this revisit
never occured; if there is exactly one solution, then we fix $x$ for this connected component, and if
it's ever fixed again through this process to a different value; declare as unsolvable. We only need
to consider this for a node we're about to revisit, and not further down the chain, as the solving
process connects those two chains of representations already. Concretely, this means that if we're
about to revisit a node, we still compute its new representation as described in the previous paragraph;
then, we set the two representations equal, to see if we need to fix $x$ for the connected component.

At the end, if there is a fixed value for a connected component, then instead of using a procedure similar
to a difference array, we just check how many ranges the single fixed value satisifies.  Doing the
above procedure for each connected component yields the answer.

## Implementation

**Time Complexity**: $\mathcal O(N+M)$

```cpp
#include <iostream>
#include <map>
#include <vector>
typedef long long ll;
using namespace std;
constexpr int MAXN = 200005;
constexpr ll SENTINEL = 9e18;
// g and w collective represents (through alignment of indices)
// the graph given by the constraints; g is the nodes that a node
// is connected to, w is the weight of the corresponding edge
// described in the same index of g
vector<int> g[MAXN];
vector<int> w[MAXN];
// As given by the problem.
ll l[MAXN], r[MAXN];
// Coefficient (m) and constant (b) representations of each node
// in terms of the reference node in each connected component.
// There *must* be a nonzero coefficient for a representation to
// be valid; therefore, the code also checks if m[v] is non-zero
// to see if a vertex has been traversed.
ll m[MAXN], b[MAXN];
// Represents fixed values for components; the index is the
// "reference" node for the component.
ll ff[MAXN];
// The minimum and maximum values of x of the connected component
// required for each specific node to function
ll minn[MAXN], maxn[MAXN];
// Keeps track of each point for a connected component.
vector<int> points[MAXN];
// A temporary map declared to facilitate maximization of nodes
// activated; see main()
map<ll, int> segments;
// A flag used to indicate if solutions cannot be found in the
// current problem; if this is set to false at any time, at the end
// of the current dataset, -1 will be printed instead of the presumed
// answer.
bool okay = true;

/**
 * Given a constraint, calculate the expression of the new node
 * in terms of the old node that is traversed from
 * @param x Specifies the old node
 * @param y Specifies the new node
 * @param z Specifies the sum required by the constraint
 * @return A pair with the coefficient and constant for the new
 *         representation
 */
pair<ll, ll> propag(int x, int y, ll z) { return make_pair(-m[x], z - b[x]); }

/**
 * A function used in situations where 2 representations
 * for a node can be computed. This function equates and
 * solve them.
 * @param m1 Coefficient for the first representation.
 * @param b1 Constant for the first representation.
 * @param m2 Coefficient for the second representation.
 * @param b2 Constant for the second representation.
 * @return A pair detailng:
 *         - first: Whether this equation is solvable with integers.
 *           If this is false, then there is no solution for the whole
 *           problem.
 *         - second.first: Whether there is any action required to
 *           fix a value.  This is meaningless if first is false.
 *         - second.second: The fixed x, if necessary, for the
 *           connected component as determined by the equation.
 */
pair<bool, pair<bool, ll>> loopback(ll m1, ll b1, ll m2, ll b2) {
	// If the coefficients are the same, the linear terms may be
	// canceled; we get b1 = b2. If that is not solvable (aka.
	// b1 != b2), report so.
	if (m1 == m2 && b1 != b2) {
		return make_pair(false, make_pair(false, 0));
	}
	// If the 2 representations are essentially the same, then there's
	// nothing to do (i.e. any x suffices).
	else if (m1 == m2 && b1 == b2) {
		return make_pair(true, make_pair(false, 0));
	}
	// The equation is m1*x+b1=m2*x+b2 --> (m1-m2)*x=(b2-b1) -->
	// x=(b2-b1)/(m1-m2).  If this is a fraction, return impossible
	// as we cannot have a fraction in any nodes (including the
	// reference.)
	else {
		if ((b2 - b1) % (m1 - m2) == 0) {
			return make_pair(true, make_pair(true, (b2 - b1) / (m1 - m2)));
		}
		return make_pair(false, make_pair(false, 0));
	}
}

/**
 * Recursive DFS routine.
 * @param label The reference node number for the connected component of
 *              the node traversed.
 * @param fa The previous node that we came from during the traversal.
 *           This is only used to avoid re-traversing to the old node we
 *           came from, so the initial call simply enters a dummy fa value.
 * @param x The current node number.
 */
void dfs(int label, int fa, int x) {
	points[label].push_back(x);
	// By this point, there is already a representation of the
	// current node; we solve m * {min, max} + b = {l, r}, so
	// {min, max} = ({l, r} - b) / m; we compute the values with
	// both l and r, and the smaller is min and the larger is max.
	// No worrying about rounding, as m[x] = +/- 1.
	minn[x] = min((l[x] - b[x]) / m[x], (r[x] - b[x]) / m[x]);
	maxn[x] = max((l[x] - b[x]) / m[x], (r[x] - b[x]) / m[x]);
	for (int i = 0; i < g[x].size(); i++) {
		int y = g[x][i];
		int z = w[x][i];
		if (y == fa) continue;
		// Build a representation for the node we are to traverse to.
		auto result = propag(x, y, z);
		ll m1 = result.first;
		ll b1 = result.second;
		// If it doesn't have a representation yet, store the representation
		// computed.
		if (m[y] == 0) {
			m[y] = m1;
			b[y] = b1;
			dfs(label, x, y);
		} else {
			// Use loopback to equate the current representation with the
			// previous representation, and act accordingly.
			ll m2 = m[y], b2 = b[y];
			auto result = loopback(m1, b1, m2, b2);
			if (!result.first) { okay = false; }
			if (result.second.first) {
				// A sentinel value indicates that we have not set a value
				// before.
				if (ff[label] == SENTINEL) {
					ff[label] = result.second.second;
				} else {
					// If there is already a set value for the traversing node,
					// then this is a conflict.
					if (ff[label] != result.second.second) { okay = false; }
				}
			}
		}
	}
}
signed main() {
	int t;
	cin >> t;
	while (t--) {
		int n, M;
		cin >> n >> M;
		for (int i = 1; i <= n; i++) {
			g[i].clear();
			w[i].clear();
			m[i] = 0;
			b[i] = 0;
			points[i].clear();
		}
		okay = true;
		for (int i = 1; i <= n; i++) cin >> l[i];
		for (int i = 1; i <= n; i++) cin >> r[i];
		for (int i = 1; i <= n; i++) ff[i] = SENTINEL;
		for (int i = 1; i <= M; i++) {
			int x, y, z;
			cin >> x >> y >> z;
			g[x].push_back(y);
			w[x].push_back(z);
			g[y].push_back(x);
			w[y].push_back(z);
		}
		for (int i = 1; i <= n; i++) {
			if (!m[i]) {
				m[i] = 1;
				dfs(i, -1, i);
			}
		}
		if (!okay) {
			cout << "-1" << endl;
			continue;
		}

		int ans = 0;
		for (int i = 1; i <= n; i++) {
			if (ff[i] == SENTINEL) {
				segments.clear();
				for (auto pt : points[i]) {
					if (minn[pt] <= maxn[pt]) {
						segments[minn[pt]]++;
						segments[maxn[pt] + 1]--;
					}
				}
				int total = 0;
				int mmax = -1;
				for (auto sgmt : segments) {
					total += sgmt.second;
					mmax = max(mmax, total);
				}
				mmax = max(mmax, total);
				ans += mmax;
			} else {
				for (auto pt : points[i]) {
					if (minn[pt] <= ff[i] && ff[i] <= maxn[pt]) { ans++; }
				}
			}
		}
		cout << ans << endl;
	}
	return 0;
}
```
