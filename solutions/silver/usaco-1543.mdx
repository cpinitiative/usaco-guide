# Solution

## Explanation

The constraints $(x, y, z)$ form an undirected weighted graph, where the constraints
are the edges and the generators are the vertices.  We optimize each component independently.

Let us first consider the cases when the component is a tree. In this case, we choose an arbitary vertex,
$v$, and let the variable $x$ represent $a_v$.

Starting from vertex $v$, we traverse the tree, defining each node's value by its parent node and edge values.
For example, if we start from node $v$ (which has a value of $x$) to node $u$ with an edge of weight $5$, we
can rewrite $a_u$ as $5 - a_v$, or $5 - x$. After traversing the tree, we can rewrite each node's value as a
function of $x$, either in the form $a - x$ or $x - a$, where $a$ is a positive constant.

For each node, after calculating its value in terms of $x$, we calculate the minimum and maximum value that $x$
can take to satisfy the stable operating range of the reactor. This reframes our problem as finding the value of
$x$ that is inside the most ranges, which can be solved using a difference array with a map.

Now, what happens if we have cycles? In our traversal for each component, cycles break the assumption that
we will not revisit nodes. When we do revisit one, we will get *two* different representations of power
mandated for a generator. We equate the expressions and solve for $x$, resulting in three cases:
- If there are no **integer** solutions, fulfilling the constraints is impossible.
- If there are **infinite** solutions, we do nothing.
- If there is **one** solution, we set the value of $x$ for this component.
  - If a fixed value is already set for this component and it is different, declare this problem unsolvable.

If there is a fixed value set for the component, then we just check how many ranges of $x$ this specific fixed
value is inside, instead of choosing an arbitrary $x$ with the most ranges covering it.

Performing the above procedure for each connected component and summing yields the answer.

## Implementation

**Time Complexity**: $\mathcal O(N+M)$

The sign of $x$ is also the coefficient, represented with $m$.  The code uses a more generalized approach
for simplicity to avoid branching for sign.

#include <iostream>
#include <map>
#include <vector>
typedef long long ll;
using namespace std;
constexpr int MAXN = 200005;
constexpr ll SENTINEL = 9e18;
// g and w represents the graph given by the constraints;
// g is the connecting node, w is the weight of the edge w/ same
// indicies in g
vector<int> g[MAXN];
vector<int> w[MAXN];
ll l[MAXN], r[MAXN];
// Stores the representation computed for each node, as
// a_v = m_v x + b.
// m is also the sign, so it's +/- 1 when it's valid.  Thus, the code
// also checks if m[v] is non-zero to see if a vertex has been traversed.
ll m[MAXN], b[MAXN];
// ff[i] is any fixed value, if set, for a component using reference node i.
ll ff[MAXN];
// Computed minimums and maximums reference values required for each generator
// to function.
ll minn[MAXN], maxn[MAXN];
// List of points for each connected component; empty for non-reference nodes.
vector<int> points[MAXN];
// A temporary map declared to facilitate maximization of nodes
// activated; see main()
map<ll, int> segments;

// A flag used to indicate if solutions cannot be found in the
// current problem; if this is set to false at any time, at the end
// of the current dataset, -1 will be printed instead of the presumed
// answer.
bool okay = true;

/**
 * Given a constraint, calculate the expression of the new node
 * in terms of the old node that is traversed from
 * @param x Specifies the old node
 * @param y Specifies the new node
 * @param z Specifies the sum required by the constraint
 * @return A pair with the coefficient and constant for the new
 *         representation
 */
pair<ll, ll> propag(int x, int y, ll z) { return make_pair(-m[x], z - b[x]); }

/**
 * A function used in situations where 2 representations
 * for a node can be computed. This function equates and
 * solve them.
 * @param m1 Coefficient for the first representation.
 * @param b1 Constant for the first representation.
 * @param m2 Coefficient for the second representation.
 * @param b2 Constant for the second representation.
 * @return A pair detailng:
 *         - first: Whether this equation is solvable with integers.
 *           If this is false, then there is no solution for the whole
 *           problem.
 *         - second.first: Whether there is any action required to
 *           fix a value.  This is meaningless if first is false.
 *         - second.second: The fixed x, if necessary, for the
 *           connected component as determined by the equation.
 */
pair<bool, pair<bool, ll>> loopback(ll m1, ll b1, ll m2, ll b2) {
	// If the coefficients are the same, the linear terms may be
	// canceled; we get b1 = b2. If that is not solvable (aka.
	// b1 != b2), report so.
	if (m1 == m2 && b1 != b2) {
		return make_pair(false, make_pair(false, 0));
	}
	// If the 2 representations are essentially the same, then there's
	// nothing to do (i.e. any x suffices).
	else if (m1 == m2 && b1 == b2) {
		return make_pair(true, make_pair(false, 0));
	}
	// The equation is m1*x+b1=m2*x+b2 --> (m1-m2)*x=(b2-b1) -->
	// x=(b2-b1)/(m1-m2).  If this is a fraction, return impossible
	// as we cannot have a fraction in any nodes (including the
	// reference.)
	else {
		if ((b2 - b1) % (m1 - m2) == 0) {
			return make_pair(true, make_pair(true, (b2 - b1) / (m1 - m2)));
		}
		return make_pair(false, make_pair(false, 0));
	}
}

/**
 * Recursive DFS routine.
 * @param label The reference node number for the connected component of
 *              the node traversed.
 * @param fa The previous node that we came from during the traversal.
 *           This is only used to avoid re-traversing to the old node we
 *           came from, so the initial call simply enters a dummy fa value.
 * @param x The current node number.
 */
void dfs(int label, int fa, int x) {
	points[label].push_back(x);
	// By this point, there is already a representation of the
	// current node; we solve m * {min, max} + b = {l, r}, so
	// {min, max} = ({l, r} - b) / m; we compute the values with
	// both l and r, and the smaller is min and the larger is max.
	// No worrying about rounding, as m[x] = +/- 1.
	minn[x] = min((l[x] - b[x]) / m[x], (r[x] - b[x]) / m[x]);
	maxn[x] = max((l[x] - b[x]) / m[x], (r[x] - b[x]) / m[x]);
	for (int i = 0; i < g[x].size(); i++) {
		int y = g[x][i];
		int z = w[x][i];
		if (y == fa) continue;
		// Build a representation for the node we are to traverse to.
		auto result = propag(x, y, z);
		ll m1 = result.first;
		ll b1 = result.second;
		// If it doesn't have a representation yet, store the representation
		// computed.
		if (m[y] == 0) {
			m[y] = m1;
			b[y] = b1;
			dfs(label, x, y);
		} else {
			// Use loopback to equate the current representation with the
			// previous representation, and act accordingly.
			ll m2 = m[y], b2 = b[y];
			auto result = loopback(m1, b1, m2, b2);
			if (!result.first) { okay = false; }
			if (result.second.first) {
				// A sentinel value indicates that we have not set a value
				// before.
				if (ff[label] == SENTINEL) {
					ff[label] = result.second.second;
				} else {
					// If there is already a set value for the traversing node,
					// then this is a conflict.
					if (ff[label] != result.second.second) { okay = false; }
				}
			}
		}
	}
}
signed main() {
	int t;
	cin >> t;
	while (t--) {
		int n, M;
		cin >> n >> M;
		// Reset any used globals that may affect the next iteration.
		for (int i = 1; i <= n; i++) {
			g[i].clear();
			w[i].clear();
			m[i] = 0;
			b[i] = 0;
			points[i].clear();
		}
		okay = true;

		for (int i = 1; i <= n; i++) cin >> l[i];
		for (int i = 1; i <= n; i++) cin >> r[i];
		// Initialize all component id fixed values to SENTINEL.
		for (int i = 1; i <= n; i++) ff[i] = SENTINEL;
		for (int i = 1; i <= M; i++) {
			int x, y, z;
			cin >> x >> y >> z;
			g[x].push_back(y);
			w[x].push_back(z);
			g[y].push_back(x);
			w[y].push_back(z);
		}

		for (int i = 1; i <= n; i++) {
			// Check if the node has been visited before,
			// since all visited nodes must have an assigned m
			// or coefficient representation.
			if (!m[i]) {
				// Initialize to =reference value=1x+0.
				m[i] = 1;
				b[i] = 0;
				dfs(i, -1, i);
			}
		}
		if (!okay) {
			cout << "-1" << endl;
			continue;
		}

		int ans = 0;
		// Process each connected component independently.
		// In here, there may be extra is enumerated
		// that are not a reference node; however, in those
		// cases, there are no points in points[i], which means
		// those will essentially be noops.
		for (int i = 1; i <= n; i++) {
			if (ff[i] == SENTINEL) {
				// No fixed value; count the reference value that
				// satisifies the most generators.
				// segments is used as a difference array, though
				// using map guarantees that only significant changes
				// are stored to save memory.
				segments.clear();
				for (auto pt : points[i]) {
					// If minn[pt] > maxn[pt], there's no range
					// to generate power for that generator, so we
					// discard it.
					if (minn[pt] <= maxn[pt]) {
						segments[minn[pt]]++;
						segments[maxn[pt] + 1]--;
					}
				}
				// Use the prefix sum approach to determine the maximum
				// covers on any point.
				int total = 0;
				int mmax = -1;
				for (auto sgmt : segments) {
					total += sgmt.second;
					mmax = max(mmax, total);
				}
				mmax = max(mmax, total);
				ans += mmax;
			} else {
				// If there is already a fixed reference value, then
				// just check if it falls into the required reference
				// value for this generator to function.
				for (auto pt : points[i]) {
					if (minn[pt] <= ff[i] && ff[i] <= maxn[pt]) { ans++; }
				}
			}
		}
		cout << ans << endl;
	}
	return 0;
}
