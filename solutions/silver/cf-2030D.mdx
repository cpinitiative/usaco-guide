## Explanation
> Throughout, $s$ is 1-indexed, so $s_k$ refers to the $k$-th character of the string.

We note that the order of swaps doesn't matter at all. Whether a value can move from one position to another depends only on $s$ , so the problem reduces to a static question: for each value, is its required path through $s$ unobstructed?

A value can cross the gap between position $k$ and position $k+1$ to the right if $s_k = R$, and to the left if $s_{k+1} = L$. The gap is impassable only when $s_k = L$ and $s_{k+1} = R$ hold simultaneously: call this a **bad gap**.

For each value $v$, let $pos$ be its current position in $p$. For $p$ to become sorted, $v$ must end up at position $v$, so it must travel from $pos$ to $v$, crossing every gap strictly between $\min(pos, v)$ and $\max(pos, v)$. We can see that there can be no bad gaps between $pos$ and $v$ for all values of $v$.

Each value's range $[\min(pos,v), \max(pos,v)]$ marks the gaps it depends on. Build a difference array: add $1$ at $\min(pos,v)$ and subtract $1$ at $\max(pos,v)$ for each value $v$. After taking prefix sums, any gap with a positive value is required by at least one value: call this an **important** gap. Gaps that are never important can be ignored regardless of whether they're bad.

### Optimal Strategy

Only important gaps matter, and a query at index $i$ can only change whether gap $i-1$ or gap $i$ is bad, since those are the only two gaps that involve $s_i$.
Keep one counter, `bad`, for how many important gaps are currently bad.

Before flipping $s_i$, check gap $i-1$ and gap $i$. If either is important and currently bad, subtract one from `bad` per bad gap, as their "bad-ness" may change. Then, flip $s_i$.
Now, once again check the same two gaps. If either is now important and bad, add one back to `bad`, per bad gap. The answer is YES only when `bad` equals zero.

## Implementation

**Time Complexity:** $\mathcal{O}(N + Q)$

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	int test_num;
	cin >> test_num;
	for (int t = 0; t < test_num; t++) {
		int n, q;
		cin >> n >> q;
		vector<int> p(n + 1), pos(n + 1);
		for (int i = 1; i <= n; i++) {
			cin >> p[i];
			pos[p[i]] = i;
		}
		string s;
		cin >> s;
		s = " " + s;  // 1-index s

		// d[k] counts how many values require gap k to stay good
		vector<int> d(n + 2, 0);
		for (int v = 1; v <= n; v++) {
			int lo = min(v, pos[v]), hi = max(v, pos[v]);
			d[lo]++;
			d[hi]--;
		}
		for (int k = 1; k <= n - 1; k++)
			d[k] += d[k - 1];  // d[k] > 0 means gap k is important

		auto is_bad = [&](int k) { return s[k] == 'L' && s[k + 1] == 'R'; };

		// bad = number of important gaps that are currently bad
		int bad = 0;
		for (int k = 1; k <= n - 1; k++)
			if (d[k] > 0 && is_bad(k)) bad++;

		while (q--) {
			int i;
			cin >> i;
			// only gaps i-1 and i can change "bad-ness" when s[i] flips
			for (int k : {i - 1, i})
				if (k >= 1 && k <= n - 1 && d[k] > 0 && is_bad(k)) bad--;
			s[i] = (s[i] == 'L') ? 'R' : 'L';
			for (int k : {i - 1, i})
				if (k >= 1 && k <= n - 1 && d[k] > 0 && is_bad(k)) bad++;
			cout << (bad == 0 ? "YES" : "NO") << "\n";
		}
	}
}
```
