## Explanation

A value can cross the gap between position $k$ and position $k+1$ to the right if $s_k = R$, and to the left if $s_{k+1} = L$. 
As long as a gap stays open, values can keep crossing it back and forth in either direction, so any sequence of swaps across 
open gaps can be rearranged into any other order without changing which values end up where. So, the order of swaps doesn't 
matter: whether a value can move from one position to another depends only on $s$, and the problem reduces to a static 
question - for each value, is its required path through $s$ unobstructed?

The gap is impassable only when $s_k = L$ and $s_{k+1} = R$ hold simultaneously: call this a **bad gap**. Bad gaps split 
the array into maximal intervals where every two adjacent positions are freely swappable, so within one such 
interval, the values can be rearranged into any order.

For each value $v$, let $pos$ be its current position in $p$. For $p$ to become sorted, $v$ must end up at position 
$v$, so it must travel from $pos$ to $v$, crossing every gap strictly between $\min(pos, v)$ and $\max(pos, v)$. 
Another way of saying this is that $pos$ and $v$ must lie in the same interval - there can be no bad gaps 
between $pos$ and $v$ for all values of $v$.

Each value's range $[\min(pos,v), \max(pos,v)]$ marks the gaps it depends on. Build a difference array: add $1$ 
at $\min(pos,v)$ and subtract $1$ at $\max(pos,v)$ for each value $v$. After taking prefix sums, any gap with a 
positive value is required by at least one value: call this an **important** gap. An **important** gap can't be 
ignored since some value $v$ needs to cross it to reach its desired position. All other gaps can be 
ignored, regardless of whether they are bad.

Only important gaps matter, since those are the only gaps that can block sorting. Maintain a counter, `bad`, 
for how many important gaps are currently bad - the answer is YES exactly when `bad` equals zero. A query 
at index $i$ only affects gap $i-1$ and gap $i$, since those are the only two gaps involving $s_i$, 
so we can update `bad` directly.

## Implementation

**Time Complexity:** $\mathcal{O}(N + Q)$

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	int test_num;
	cin >> test_num;
	for (int t = 0; t < test_num; t++) {
		int n, q;
		cin >> n >> q;
		vector<int> p(n + 1), pos(n + 1);
		for (int i = 1; i <= n; i++) {
			cin >> p[i];
			pos[p[i]] = i;
		}
		string s;
		cin >> s;
		s = " " + s;  // 1-index s

		// d[k] counts how many values require gap k to stay good
		vector<int> d(n + 1);
		for (int v = 1; v <= n; v++) {
			int lo = min(v, pos[v]), hi = max(v, pos[v]);
			d[lo]++;
			d[hi]--;
		}
		for (int k = 1; k <= n - 1; k++) {
			d[k] += d[k - 1];  // d[k] > 0 means gap k is important
		}

		auto is_bad = [&](int k) { return s[k] == 'L' && s[k + 1] == 'R'; };

		// bad = number of important gaps that are currently bad
		int bad = 0;
		for (int k = 1; k <= n - 1; k++) {
			if (d[k] > 0 && is_bad(k)) {
				bad++;
			} 
		}
			

		while (q--) {
			int i;
			cin >> i;
			// only gaps i-1 and i can change "bad-ness" when s[i] flips
			for (int k : {i - 1, i}) {
				if (k >= 1 && k <= n - 1 && d[k] > 0 && is_bad(k)) {
					bad--;
				} 
			}
			s[i] = (s[i] == 'L') ? 'R' : 'L';
			for (int k : {i - 1, i}) {
				if (k >= 1 && k <= n - 1 && d[k] > 0 && is_bad(k)) {
					bad++;
				} 
			}
			cout << (bad == 0 ? "YES" : "NO") << "\n";
		}
	}
}
```
