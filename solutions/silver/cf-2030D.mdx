## Explanation
> Throughout, $s$ is 1-indexed, so $s_k$ refers to the $k$-th character of the string.

Whether a value can get from one position to another depends only on $s$. The order of swaps doesn't matter at all.
So the real question becomes: which gaps in $s$ are "blocked", and whether any of those blocked gaps fall under positions where the permutation needs them to be open?
**Gap $k$ depends only on $s_k$ and $s_{k+1}$, so flipping $s_i$ can only affect gap $i-1$ and gap $i$. Hence, we never need to recheck everything from scratch.**

Now, look at the gap between position $k$ and position $k+1$. A value can cross this gap to the right if $s_k = R$, and to the left if $s_{k+1} = L$.
The only time a gap can **NOT** be crossed is when $s_k = L$ and $s_{k+1} = R$ at the same time. Call this a **bad gap**.

For each value $v$, let $pos$ be its current position in $p$. For the array to end up sorted, $v$ must travel between $pos$ and $v$.
WLOG, let $pos$ be the smaller of the 2 values, and $v$ be the larger. We can see that there can be no **bad** gaps between $pos$ and $v$ for all values of $v$.

So naturally, we ask: How do we track these ranges?

The trick is to use a difference array: for each value's range $[lo, hi]$, add $1$ at $lo$ and subtract $1$ at $hi$. After taking prefix sums, any gap with a positive value is needed by at least one value. Call these the **important gaps**.
We can effectively ignore all other gaps.

### Optimal Strategy

Only important gaps matter, and a query at index $i$ can only change whether gap $i-1$ or gap $i$ is bad, since those are the only two gaps that involve $s_i$.
Keep one counter, `bad`, for how many important gaps are currently bad.

Before flipping $s_i$, check gap $i-1$ and gap $i$. If either is important and currently bad, subtract one from `bad` per bad gap, as their "bad-ness" may change. Then, flip $s_i$.
Now, once again check the same two gaps. If either is now important and bad, add one back to `bad`, per bad gap. **The answer is YES when `bad` equals zero.**

## Implementation

**Time Complexity:** $\mathcal{O}(N + Q)$

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
	int t;
	cin >> t;
	while (t--) {
		int n, q;
		cin >> n >> q;
		vector<int> p(n + 1), pos(n + 1);
		for (int i = 1; i <= n; i++) {
			cin >> p[i];
			pos[p[i]] = i;
		}
		string s;
		cin >> s;
		s = " " + s;  // 1-index s

		// d[k] counts how many values require gap k to stay good
		vector<int> d(n + 2, 0);
		for (int v = 1; v <= n; v++) {
			int lo = min(v, pos[v]), hi = max(v, pos[v]);
			d[lo]++;
			d[hi]--;
		}
		for (int k = 1; k <= n - 1; k++)
			d[k] += d[k - 1];  // d[k] > 0 means gap k is important

		auto isbad = [&](int k) { return s[k] == 'L' && s[k + 1] == 'R'; };

		// bad = number of important gaps that are currently bad
		int bad = 0;
		for (int k = 1; k <= n - 1; k++)
			if (d[k] > 0 && isbad(k)) bad++;

		while (q--) {
			int i;
			cin >> i;
			// only gaps i-1 and i can change "bad-ness" when s[i] flips
			for (int k : {i - 1, i})
				if (k >= 1 && k <= n - 1 && d[k] > 0 && isbad(k)) bad--;
			s[i] = (s[i] == 'L') ? 'R' : 'L';
			for (int k : {i - 1, i})
				if (k >= 1 && k <= n - 1 && d[k] > 0 && isbad(k)) bad++;
			cout << (bad == 0 ? "YES" : "NO") << "\n";
		}
	}
}
```
