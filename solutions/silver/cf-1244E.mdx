## Explanation

Instead of trying to construct a final array, fix a target difference $d$ and check if it is achievable in under $k$ operations.

If all elements can be moved into some interval of length $d$, then it can be shown that all larger values of $d$ work. Hence, the condition is monotonic and we can binary search on $d$.

For a fixed $d$, we try to fit all elements into $[L, L + d]$, where $L$ is an arbitrary starting point for the interval:
- increase elements below $L$
- decrease elements above $L + d$

The total cost (number of unit increments/decrements needed to move all elements into [L,L+d]) can be computed using prefix sums.

### Choosing L

We consider an interval [L, L + d] of fixed length d, where L is the left endpoint.

If we move this interval slightly to the right without crossing any a[i], then:
- elements left of L become more expensive to increase
- elements right of L + d become cheaper to decrease

So we can always keep moving in the direction that does not increase cost until we hit some a[i].

Therefore, an optimal interval must have at least one endpoint at some a[i], i.e.:

L = a[i] or L + d = a[i]

### Calculating the Final Answer

For a fixed L, use prefix sums $pref[i]$.

Let $cnt$ = number of elements with a[i] < L.

Left cost:
$(L - a[1]) + ... + (L - a[cnt])$
= $cnt * L - \text{pref}[cnt]$

Let $right$ = last index with a[right] <= L + d.

Right cost:
$(a[right+1] - (L + d)) + ... + (a[n] - (L + d))$
= $(\text{pref}[n] - \text{pref}[right]) - (n - right) * (L + d)$

Total cost:
$(cnt * L - \text{pref}[cnt]) + (\text{pref}[n] - \text{pref}[right] - (n - right) * (L + d))$

A value of $d$ is feasible if the minimum cost over all choices of L is ≤ k.

$cnt$ and $right$ are found via binary search, so each L is evaluated in O(log n).
We take the minimum over all candidate L.


## Implementation

**Time Complexity:** $\mathcal{O}(N \log N \log A)$, where $A$ is the maximum value of $a_i$.

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	int n;
	ll k;
	cin >> n >> k;

	vector<int> a(n);
	for (int &x : a) cin >> x;
	sort(a.begin(), a.end());

	// compute prefix sums, using 1 indexing for convenience
	vector<ll> pref(n + 1);
	for (int i = 0; i < n; i++) { pref[i + 1] = pref[i] + a[i]; }

	auto check = [&](int d) -> bool {
		for (int i = 0; i < n; ++i) {
			// Case 1: l = a[i] (left endpoint pinned to a[i])
			{
				int l = a[i], r = l + d;

				// use prefix sums to calculate l - a_i for all a_i < l
				ll cost_l = 1ll * i * l - pref[i];

				// use prefix sums to calculate a_i - r for all a_i > r
				// we use upper bound to find the first index we sum from
				int p = upper_bound(a.begin(), a.end(), r) - a.begin();
				ll cost_r = (pref[n] - pref[p]) - 1ll * (n - p) * r;

				if (cost_l + cost_r <= k) return true;
			}

			// Case 2: l = a[i] - d (right endpoint pinned to a[i])
			{
				int r = a[i], l = r - d;

				// use prefix sums to calculate a_i - r for all a_i > r
				ll cost_r = (pref[n] - pref[i + 1]) - 1ll * (n - i - 1) * r;

				// use prefix sums to calculate l - a_i for all a_i < l
				// we use lower bound to find the last index we sum from
				int p = lower_bound(a.begin(), a.end(), l) - a.begin();
				ll cost_l = 1ll * p * l - pref[p];

				if (cost_l + cost_r <= k) return true;
			}
		}

		return false;
	};

	int lo = 0, hi = a[n - 1] - a[0];
	while (lo < hi) {
		int mid = lo + (hi - lo) / 2;
		if (check(mid)) {
			hi = mid;
		} else {
			lo = mid + 1;
		}
	}

	cout << lo << '\n';
}
```
