<Spoiler title="Hint 1">

The final gcd can only be affected by primes that currently divide $\gcd(a_1, \ldots, a_n)$ — call these the **target primes**. How many such primes can there be?

</Spoiler>

<Spoiler title="Hint 2">

Since there are at most 11 target primes, represent the set of target primes a judge can eliminate in one game as a bitmask. What's the minimum divisor needed to eliminate a specific subset of target primes from a judge's doubt?

</Spoiler>

<Spoiler title="Hint 3">

With $n$ up to $10^6$, running the DP once per judge is too slow. Two judges are completely interchangeable in the DP if they offer the same set of "maximal achievable masks." How can you collapse $10^6$ judges down to a tiny set?

</Spoiler>

## Explanation

Our first key observation is that the only way that the final $\gcd$ can remain $> 1$ is if some prime $p$ divides every $a_i$. So the only primes that can block us are those dividing $g = \gcd(a_1, \ldots, a_n)$.

This means we can ignore all the other primes: any prime not dividing $g$ already fails to divide some $a_i$, so it cannot contribute to the final $\gcd$.

If $g = 1$ then, the answer is $0$ immediately as no games are needed.
Otherwise, factor $g$, and let $m$ be the number of distinct prime factors of $g$. Let $\text{primes}[0], \ldots, \text{primes}[m-1]$ denote each of the $m$ prime factors.
Define $\text{pw}[j]$ for judge $i$ as the **full prime power** of $\text{primes}[j]$ in $a_i$:

$$
\text{pw}[j] = p_j^{v_{p_j}(a_i)}, \quad p_j = \text{primes}[j]
$$

For example: If $a_i = 12 = 2^2 \cdot 3$, then $\text{primes}[j] = 2$ and $\text{pw}[j] = 4$.

We note that $m \leq 11$, since the product of the first 12 primes exceeds $10^{12}$.

Since $g \mid a_i$ for all $i$, every prime in $\text{primes}$ divides every $a_i$. However, dividing isn't enough to remove a prime in one game, as you can only knock out $\text{primes}[j]$ from judge $i$ if your chosen divisor is at least $\text{pw}[j]$, the full prime power of $\text{primes}[j]$ in $a_i$.

Therefore:
- To clear prime $j$ from judge $i$, you need divisor $d$ with $\text{pw}[j] \mid d$, i.e. $d \geq \text{pw}[j]$.
- To clear a subset $S$ of primes simultaneously, you need $d$ divisible by all $\text{pw}[j]$ for $j \in S$. Since the target primes are distinct, these are coprime, so the minimum such $d$ is $\prod_{j \in S} \text{pw}[j]$.

If this product exceeds $k$, the subset $S$ is impossible for this judge in one game.

First, since $m \leq 11$, we represent the subsets of target primes as bitmasks (bit $j$ being toggled = $\text{primes}[j]$ is in the subset).
Then, for each judge $i$ and bitmask $s$, the minimum divisor to clear subset $s$ in one game is:

$$
\text{prod}[s] = \prod_{j \in s} \text{pw}[j]
$$

If $\text{prod}[s] \leq k$, judge $i$ can clear all primes in $s$ in a single game.

So the problem reduces to choosing a set of judges such that the union of their cleared prime sets equals all $m$ target primes $(\text{full} = (1 \ll m) - 1)$, minimizing $(\text{count}) \times (\text{sum of experience})$.

### Step 1: Efficient Subset-Product Computation

Computing $\text{prod}[s]$ naively for each subset is $O(2^m \cdot m)$ per judge. But we can optimise our code to compute it in $O(2^m)$:

Let $t$ be $s$ with its lowest set bit (index $j$) cleared. We can compute $\text{prod}[s]$ from $\text{prod}[t]$ by multiplying by $\text{pw}[j]$: $\text{prod}[s] = \text{prod}[t] \times \text{pw}[j]$. Since we iterate $s$ upward from $1$, $t$ (being smaller than $s$) is always already computed.

<LanguageSection>

<CPPSection>

```cpp
prod[0] = 1;
for (int s = 1; s < full; ++s) {
	int j = __builtin_ctz(s);             // index of lowest set bit
	long long prev = prod[s ^ (s & -s)];  // prod[s without lowest bit]
	prod[s] = (prev > k / pw[j]) ? (k + 1) : prev * pw[j];
}
```

</CPPSection>

</LanguageSection>

<Info title="Overflow Guard">

Both `prev` and `pw[j]` can reach $10^{12}$, so their product may overflow `long long`. Instead of computing the product, check `prev > k / pw[j]` first. If the product exceeds $k$, store a sentinel value `k+1`, as any value exceeding k makes no difference.

</Info>

### Step 2: Maximal Achievable Masks (Pruning)

There are at most $2^11=2048$ possible sets $s$, and we aim to optimize that number to make our processing of judges faster.  We note that there are many sets $s$ that are not neccessary to result in an optimal solution.

For a given judge, call a subset $s$ **affordable** if $\text{prod}[s] \leq k$. Many affordable subsets are dominated. If a subset $s$ and its strict superset $s' \supset s$ for a given judge are both affordable, there's never any reason to clear just $s$: you could clear $s'$ instead for the same cost, getting strictly more coverage.

So we only care about **maximal** affordable subsets: those where no affordable strict superset exists.

<LanguageSection>

<CPPSection>

```cpp
vector<int> opts;
for (int s = 1; s < full; ++s) {
	if (prod[s] > k) continue;  // subset s is unaffordable, skip
	bool maximal = true;
	for (int j = 0; j < m && maximal; ++j)
		// if we can add prime j to s and still afford it, s is dominated
		if (!(s & (1 << j)) && prod[s | (1 << j)] <= k) maximal = false;
	if (maximal) opts.push_back(s);
}
```

</CPPSection>

</LanguageSection>

These are the only meaningful choices this judge offers. Note that `opts` can never have too many elements, since no subset in it can contain another - by Sperner's Theorem, at most $\binom{m}{\lfloor m/2 \rfloor} \leq \binom{11}{5} = 462$ elements.
For more info, check out [Sperner's Theorem.](https://en.wikipedia.org/wiki/Sperner%27s_theorem)

### The DP

The cost is $c \times \sum e_i$ where $c$ is the number of judges used. Sadly, we can't just track $\min(\sum e_i)$, since we also need to know $c$, as two
solutions covering the same prime mask with different judges have different costs.

**State:** $dp[c][s]$ represents minimum sum of experience using **exactly** $c$ judges covering prime-mask $s$.

**Base case:** $dp[0][0] = 0$, everything else $= \infty$.

**Transition:** For each judge with options list `opts`, for each option $T \in \text{opts}$:

$$
dp[c+1][s \mid T] = \min(dp[c+1][s \mid T],\ dp[c][s] + e_i)
$$

**Answer:** $\min_{c=1}^{m}\ c \times dp[c][\text{full}-1]$

Why cap at $c = m$? Any judge that doesn't contribute at least one new prime bit is purely wasteful - both $c$ and $\sum e$ increase for zero coverage gain.
Thus, with $m$ target primes, you never need more than $m$ judges.

To prevent the double-use of the same judge, iterate $c$ **downward** from $m-1$ to $0$. In each transition, we read from $dp[c]$ and write to $dp[c+1]$.
Since $c+1 > c$ and we iterate downward, no read ever sees a value written by the current judge's own pass.
This is the standard 0/1 knapsack backward-iteration trick.

### How to Optimize a Naive DP to Prevent TLE

The naive approach runs the DP update once per judge, with a time complexity of $O(n \cdot m \cdot 2^m \cdot |\text{opts}|)$.
With $n = 10^6$, $m = 11$, $2^m = 2048$, and $|\text{opts}| \leq 462$, $10^6 \times 11 \times 2048 \times 462 \approx 10^{13}$, which is completely infeasible. Thus, we aim to optimize the number of judges.

**Key reduction:** Two judges are completely interchangeable in the DP iff they have the same `opts` list, since the DP only cares about which transitions are available (`opts`) and the cost ($e_i$). So among judges with the same opts, only cost distinguishes them. And since we never use more than $m$ judges total, we never need more than $m$ judges of any single type.  So, for each unique `opts` type, we keep only the $m$ cheapest experience values.

This reduction happens in two layers:


1. **Group by exponent pattern.** Two judges with identical prime-power vectors ($\text{pw}[0], \ldots, \text{pw}[m-1]$) produce identical `prod[]` tables and identical `opts`, so we encode each judge's exponent vector into a hash (6 bits per exponent, enough since the smallest prime is 2 and $2^{40} > 10^{12}$ means no exponent exceeds 39) and group judges with matching hashes together, keeping only the $m$ cheapest experience values per group in a max-heap.
2. **Group by opts.** Even across different exponent patterns, two groups can produce the same `opts` list. These groups are interchangeable in the DP.
Use `opts` itself as the second-level key, and again keep only the $m$ cheapest experience values per type, merging across all exponent groups sharing the same `opts`.

After both optimizations, at most $\binom{11}{5} = 462$ distinct opt-types exist, each with at most $m = 11$ virtual judges - so at most $462 \times 11 \approx 5082$ judges ever touch the DP.
The final DP is uses approximately $5082 \times 11 \times 2048 \times |\text{opts}| \approx 5.7 \times 10^8$ operations, which passes within the time limit.

## Implementation

**Time Complexity:** $\mathcal{O}(N \cdot M + 2^M \cdot M^2 \cdot \binom{M}{\lfloor M/2 \rfloor})$, where $M$ is the number of distinct prime factors of $\gcd(a_1, \ldots, a_n)$ (at most 11)

<CPPSection>

```cpp
#include <bits/stdc++.h>
using namespace std;

constexpr long long INF = 1e18;

long long gcd(long long a, long long b) {
	if (b == 0) { return a; }
	return gcd(b, a % b);
}

// hash function for vector<int> keys in unordered_map
struct vector_int_hash {
	size_t operator()(const vector<int> &v) const {
		size_t h = 0;
		for (int x : v) { h = h * 1000003 ^ (unsigned)x; }
		return h;
	}
};

// hash function for (uint64_t, int) pairs used as exponent hashes
struct pair_hash {
	size_t operator()(pair<uint64_t, int> p) const {
		return hash<uint64_t>{}(p.first) ^ ((size_t)(unsigned)p.second * 2654435761ULL);
	}
};

// Used for group judges by identical exponent vectors (same prime-power structure)
// keep only the m cheapest costs per pattern (others are useless for DP)
struct exp_group {
	// pw[j] = full power of target_primes[j] in a[i]
	vector<long long> prime_powers;
	// max-heap of the m cheapest experience values seen
	vector<long long> cheap_heap;
};

// Used to convert opt_groups into a list of virtual judges for the DP
// each (opts, exp) behaves like a 0/1 knapsack item
struct virtual_judge {
	vector<int> opts;
	long long exp;
};

int main() {
	ios::sync_with_stdio(false);
	cin.tie(nullptr);

	int n;
	long long k;
	cin >> n >> k;

	vector<long long> a(n), e(n);
	for (auto &x : a) { cin >> x; }
	for (auto &x : e) { cin >> x; }

	// only prime factors of gcd(a[]) can ever appear in the final gcd
	long long g = a[0];
	for (int i = 1; i < n; ++i) { g = gcd(g, a[i]); }
	if (g == 1) {
		cout << 0;
		return 0;
	}  // already sorted, no games needed

	// factor g to get the target primes we need to eliminate
	vector<long long> target_primes;
	long long tmp = g;
	for (long long i = 2; i * i <= tmp; ++i) {
		if (tmp % i == 0) {
			target_primes.push_back(i);
			while (tmp % i == 0) { tmp /= i; }
		}
	}
	if (tmp > 1) { target_primes.push_back(tmp); }

	int m = (int)target_primes.size();
	int full_mask = (1 << m) - 1;  // bitmask with all m primes set

	// optimization 1: group judges by their exponent hash
	unordered_map<pair<uint64_t, int>, exp_group, pair_hash> exp_groups;
	exp_groups.reserve(1 << 14);

	// reused each iteration for current judge's pw[]
	vector<long long> prime_powers(m);
	for (int i = 0; i < n; ++i) {
		long long val = a[i];
		uint64_t key64 = 0;
		int key_last = 0;

		for (int j = 0; j < m; ++j) {
			long long p = target_primes[j];
			int exponent = 0;
			long long full_power = 1;
			while (val % p == 0) {
				val /= p;
				++exponent;
				full_power *= p;
			}
			prime_powers[j] = full_power;
			if (j < 10) key64 |= (uint64_t)exponent << (6 * j);
			else
				key_last = exponent;  // 11th prime overflows 64 bits, stored separately
		}

		auto key = make_pair(key64, key_last);
		auto it = exp_groups.find(key);
		if (it == exp_groups.end()) {
			exp_groups.emplace(key, exp_group{prime_powers, {}});
			it = exp_groups.find(key);
		}

		auto &heap = it->second.cheap_heap;
		heap.push_back(e[i]);
		push_heap(heap.begin(), heap.end());
		if ((int)heap.size() > m) {
			pop_heap(heap.begin(), heap.end());
			heap.pop_back();
		}
	}

	// optimization 2: group exponent groups by their opts vector,
	// keeping the m cheapest costs per opts-class
	unordered_map<vector<int>, vector<long long>, vector_int_hash> opt_groups;
	opt_groups.reserve(1 << 10);

	vector<long long> subset_prod(1 << m);
	for (auto &[_, grp] : exp_groups) {
		const vector<long long> &pw = grp.prime_powers;

		subset_prod[0] = 1;
		for (int s = 1; s < (1 << m); ++s) {
			int j = __builtin_ctz(s);
			long long prev = subset_prod[s ^ (s & -s)];
			subset_prod[s] = (prev > k / pw[j]) ? (k + 1) : prev * pw[j];
		}

		vector<int> opts;
		for (int s = 1; s < (1 << m); ++s) {
			if (subset_prod[s] > k) continue;
			bool is_maximal = true;
			for (int j = 0; j < m && is_maximal; ++j)
				if (!(s & (1 << j)) && subset_prod[s | (1 << j)] <= k)
					is_maximal = false;
			if (is_maximal) opts.push_back(s);
		}
		if (opts.empty()) continue;

		vector<long long> sorted_exp = grp.cheap_heap;
		sort(sorted_exp.begin(), sorted_exp.end());
		auto &bucket = opt_groups[opts];
		for (long long exp_val : sorted_exp) {
			bucket.push_back(exp_val);
			push_heap(bucket.begin(), bucket.end());
			if ((int)bucket.size() > m) {
				pop_heap(bucket.begin(), bucket.end());
				bucket.pop_back();
			}
		}
	}

	vector<virtual_judge> virtual_judges;
	for (auto &[opts, heap] : opt_groups) {
		vector<long long> sorted_exp = heap;
		sort(sorted_exp.begin(), sorted_exp.end());
		for (long long exp_val : sorted_exp) virtual_judges.push_back({opts, exp_val});
	}

	// dp[c][s] = min sum of experience using exactly c judges covering prime mask s
	vector<vector<long long>> dp(m + 1, vector<long long>(1 << m, INF));
	dp[0][0] = 0;

	for (auto &[opts, exp] : virtual_judges) {
		// iterate c downward to prevent reusing the same virtual judge twice (0/1
		// knapsack)
		for (int c = m - 1; c >= 0; --c) {
			for (int s = 0; s < (1 << m); ++s) {
				if (dp[c][s] == INF) continue;
				for (int T : opts) {
					long long new_exp_sum = dp[c][s] + exp;
					int new_mask = s | T;
					if (new_exp_sum < dp[c + 1][new_mask])
						dp[c + 1][new_mask] = new_exp_sum;
				}
			}
		}
	}

	// answer is min over all judge counts c of c * (sum of experience)
	long long ans = INF;
	for (int c = 1; c <= m; ++c)
		if (dp[c][full_mask] < INF) ans = min(ans, (long long)c * dp[c][full_mask]);

	cout << (ans == INF ? -1 : ans);
	return 0;
}
```
