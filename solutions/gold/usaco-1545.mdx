## Explanation
When a new party is established, one of the following happens:
1. It forms a completely new group of cows where all of them previously didn't attend any party.
2. It takes some cows from another party. If we think of this as each party having a set of guests attending, a new party forces a set to be split into two.

Efficiently splitting a set into two is difficult to implement, as we need to figure out precisely how to split it while updating counts. However, if we were to process each query in reverse, then it turns into picking the correct two sets and uniting them together, which is much easier to implement.

A data structure that allows us to pick two sets and merge them together is the Disjoint Set Union.
Each time a cow starts hosting a party (not simply changing type), when processed in reverse, we merge that cow with their best friend to join the two sets together.
Every cow in that particular party will travel to wherever the host cow goes, so we only need to worry about merging.

We can use a Disjoint Set Union to represent the friendships of the cows, where each cow is a node.
We start by computing how many cows attend each type of party at the last night and work backwards from there.
Starting from the beginning, we process all changes to parties hosted by the cows, whether that is a cow changing their party type or a cow starting a completely new party.
We use an array to keep track of the type of party each cow hosts.
After that, since a cow without a party travels to their best friend, we find all cows without a party and merge each of them with their best friend in a DSU.
Lastly, for each cow hosting a party, the `size()` method lets us find the number of cows attending their party.
Since we kept track of the type of party they are hosting, we add the amount to the appropriate array.

We work backwards from here to solve each query.
After a query is completed, we determine whether the cow had started a completely new party or only changed party type.
If it is the latter, we update the party type of that cow.
Using the `size()` function, we figure out the number of cows attending that particular party. In the array storing the number of cows attending each type of party, for each element, we add or subtract the size as necessary.
We continue until all queries are computed.

## Implementation

**Time Complexity:** $\mathcal{O}((N+M) \alpha(N))$

#include <bits/stdc++.h>
using namespace std;

// BeginCodeSnip{Disjoint Set Union}

class DisjointSets {
  private:
	vector<int> parents;
	vector<int> sizes;

  public:
	DisjointSets(int size) : parents(size), sizes(size, 1) {
		for (int i = 0; i < size; i++) { parents[i] = i; }
	}

	/** @return the "representative" node in x's component */
	int find(int x) { return parents[x] == x ? x : (parents[x] = find(parents[x])); }

	/** @return whether the merge changed connectivity */
	bool unite(int x, int y) {
		int x_root = find(x);
		int y_root = find(y);
		if (x_root == y_root) { return false; }

		if (sizes[x_root] < sizes[y_root]) { swap(x_root, y_root); }
		sizes[x_root] += sizes[y_root];
		parents[y_root] = x_root;
		return true;
	}

	/** @return whether x and y are in the same connected component */
	bool connected(int x, int y) { return find(x) == find(y); }

	/** @return the size of x's component */
	int size(int x) { return sizes[find(x)]; }
};

// EndCodeSnip
int main() {
	cin.tie(0)->sync_with_stdio(0);
	int n;
	cin >> n;
	vector<int> frnd(n + 1);          // each cow's best friend
	vector<char> hosted(n + 1, '0');  // what party each cow is hosting
	for (int i = 1; i <= n; i++) { cin >> frnd[i]; }
	int m;
	cin >> m;
	vector<pair<int, char>> party(m);
	for (int i = 0; i < m; i++) {
		int cow;
		cin >> cow;
		char type;
		cin >> type;
		party[i] = {cow, hosted[cow]};
		hosted[cow] = type;
	}
	// create DSU graph of friends
	DisjointSets graph(n);
	for (int i = 1; i <= n; i++) {
		if (hosted[i] == '0') { graph.unite(i, frnd[i]); }
	}
	// first, we find out how many cows are at each type of party at the last night
	// so we can work backwards
	vector<int> typeC(m), typeO(m), typeW(m);
	for (int i = 1; i <= n; i++) {
		switch (hosted[i]) {
		case 'C':
			typeC[m - 1] += graph.size(i);
			break;
		case 'O':
			typeO[m - 1] += graph.size(i);
			break;
		case 'W':
			typeW[m - 1] += graph.size(i);
			break;
		}
	}
	// work backwards
	for (int i = m - 1; i > 0; i--) {
		// start w/ numbers from the next time
		typeC[i - 1] = typeC[i];
		typeO[i - 1] = typeO[i];
		typeW[i - 1] = typeW[i];
		int cow = party[i].first;  // which cow is hosting the new party
		// subtract
		switch (hosted[cow]) {
		case 'C':
			typeC[i - 1] -= graph.size(cow);
			break;
		case 'O':
			typeO[i - 1] -= graph.size(cow);
			break;
		case 'W':
			typeW[i - 1] -= graph.size(cow);
			break;
		}
		hosted[cow] = party[i].second;
		char prevParty =
		    hosted[cow];  // prevParty: where the new party drew customers from
		// prevParty = '0' means new party was created, so we find where this cow's best
		// friend
		if (prevParty == '0') { prevParty = hosted[graph.find(frnd[cow])]; }
		// add
		switch (prevParty) {
		case 'C':
			typeC[i - 1] += graph.size(cow);
			break;
		case 'O':
			typeO[i - 1] += graph.size(cow);
			break;
		case 'W':
			typeW[i - 1] += graph.size(cow);
			break;
		}
		// if this cow no longer hosts a party, unite
		if (hosted[cow] == '0') { graph.unite(cow, frnd[cow]); }
	}
	// print out answer
	for (int i = 0; i < m; i++) {
		cout << typeC[i] << ' ' << typeO[i] << ' ' << typeW[i] << '\n';
	}
}
