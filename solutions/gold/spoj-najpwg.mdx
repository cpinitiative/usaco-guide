## Explanation

The given bound $N$ in each case only bounds the larger value in the pair. Thus, if we can efficiently precompute the number of valid $x < n$ for each larger value $n$, we can efficiently answer the queries.
The queries can be answered in $\mathcal{O}(1)$ by performing prefix sums of $f(n)$, where $f(n) = n - \phi(n)$. We take the complement of $\phi(n)$ because $\phi(n)$ counts number of values $x \le n$ with $\gcd(x, n)=1$, but we want the values of $x$ where $\gcd(x,n) > 1$, which are the rest of the values.

## Implementation

**Time Complexity:** $\mathcal{O}(N\log\log N + N)$

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1e5;

int phi[MAXN + 1];

void precompute() {
	for (int i = 1; i <= MAXN; i++) { phi[i] = i; }

	for (int i = 2; i <= MAXN; i++) {
		// If i is prime
		if (phi[i] == i) {
			for (int j = i; j <= MAXN; j += i) { phi[j] -= phi[j] / i; }
		}
	}

	for (int i = 1; i <= MAXN; i++) {
		// Add the pairs with gcd > 1 which is i - phi[i]
		phi[i] = phi[i - 1] + (i - phi[i]);
	}
}

int main() {
	int t;
	cin >> t;

	precompute();

	for (int i = 1; i <= t; i++) {
		int n;
		cin >> n;
		cout << "Case " << i << ": " << phi[n] << '\n';
	}
}
```
