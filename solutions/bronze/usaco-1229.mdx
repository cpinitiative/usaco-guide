## Explanation

Bessie the cow wants to maximise her units of metal $N$, for which
she can use the recipes provided to convert other constituent
metals into metal $N$.

We can recursively check the $N$-th metal constituents, and if it
has enough availability we will consume one unit of it and check
for other metal constituents. We will repeat this until we run out
of supply for constituent metals.

This recursive greedy approach works perfectly for this problem for
the following reasons:

1. Since there is only one recipe for each metal, constructing a
new unit of any metal is a binary choice: either we consume from
our existing stock, or we construct a new unit from its recipe.

2. We can always delay constructing a new unit of a metal, so we
should prioritize using our existing stock first and build new
units of metal only if necessary.

3. The ordering of recipes forms a tree-like structure, so a
recursive approach is easier to implement than other approaches.


## Implementation

**Time Complexity:** $\mathcal{O}(2^{max\_depth\_of\_recipe\_chain})$

```cpp
#include <functional>
#include <iostream>
#include <unordered_map>
#include <vector>

int main() {

	int n;
	std::cin >> n;
	std::vector<int> metals(n);
	for (int &x : metals) { std::cin >> x; }

	int k;
	std::cin >> k;
	std::vector<std::vector<int>> recipes(n);

	for (int i = 1; i <= k; i++) {
		int l, m;
		std::cin >> l >> m;
		for (int j = 1; j <= m; j++) {
			int x;
			std::cin >> x;
			recipes[l - 1].push_back(x - 1);
		}
	}

	std::function<bool(int)> possible = [&](int metal) -> bool {
		if (metals[metal] > 0) {
			metals[metal]--;
			return true;
		}

		if (recipes[metal].size() == 0) { return false; }

		for (int constituent_metals : recipes[metal]) {
			if (!possible(constituent_metals)) { return false; }
		}
		return true;
	};

	int res = 0;
	while (possible(n - 1)) { res++; }

	std::cout << res << '\n';
}
```
